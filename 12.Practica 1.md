# 2. Generar una clave SSH local

Se debe generar un par de claves criptográficas SSH (llave privada) mediante Terraform utilizando el proveedor `tls`.

2.1. Crear el archivo `variables.tf` para definir el nombre del archivo de la llave y el nombre del bucket de S3. Ambos valores se deben solicitar desde la terminal (sin valores por defecto):

Las variables no tienen valor `default`

```tf
variable "key_name", tipo  string, descripcion "Nombre del archivo pem que guardará la clave SSH privada"
variable "bucket_name", tipo string, descripcion "Nombre único a nivel global para el bucket de S3"
variable "student_name", tipo string, descripcion "Nombre del estudiante para identificar la VPC"
```
> [!TIP]
> Para repasar cómo declarar variables sin valor por defecto, consultar la guía de [Variables Input](file:///Users/car/Documents/Code/terraformlabs/05.Variables%20Input.md).

2.2. Crear el archivo `main.tf` y configurar la generación de la clave SSH y su almacenamiento en un archivo local:

```tf
# Generación de la clave criptográfica privada
resource "tls_private_key", algorithm = "RSA", rsa_bits  = 4096

# Almacenamiento de la clave privada en un archivo local
resource "local_sensitive_file", content = [ASIGNAR EL CONTENIDO DE LA LLAVE EN FORMATO PEM], filename = [USAR LA VARIABLE key_name y concatenar extension .pem]
```
> [!TIP]
> Para repasar el proveedor `tls` y el recurso `local_sensitive_file`, consultar la guía de [Atributos](file:///Users/car/Documents/Code/terraformlabs/06.Atributos.md).

# 4. Almacenamiento seguro en S3

Se debe crear un bucket de S3 privado en AWS y subir el archivo de clave criptográfica generado.

4.1. Crear el bucket en `main.tf`:
```tf
resource "aws_s3_bucket", nombre = [USAR LA VARIABLE bucket_name]
```

4.2. Cargar la clave SSH privada en el bucket creado:
```tf
resource "aws_s3_object", key    = ["${var.key_name}.pem"] , source = [ASIGNAR EL CONTENIDO DE LA LLAVE EN FORMATO PEM DEL PUNTO 2.2]
```

7.4. **Autoevaluación (Checkpoint)**:
Una vez completado con éxito, se pueden verificar los recursos en la nube mediante AWS CLI ejecutando los siguientes comandos en la terminal (reemplazando por los nombres asignados):
