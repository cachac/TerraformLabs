```tf
resource "azurerm_resource_group" "main" {

# Llave SSH Nativa de Terrafrom

```

### 4.2.2. Subnet Publica B

## 4.3. Instaurar NAT Gateway
A diferencia de AWS que requiere múltiples tablas y portazos entre Subnets para dar salida de internet, en Azure basta con atar la Subred directamente al objeto "NAT Gateway". Todas las peticiones saldrán mágicamente ahí.
```tf
# 1. Creamos IP estática oficial para el NAT
resource "azurerm_public_ip" "nat_ip" {
  name                = "nat-ip"
  location            = var.location
  resource_group_name = var.resource_group_name
  allocation_method   = "Static"
  sku                 = "Standard"
}

# 2. Fabricamos el router de Salida (Router de Máscaras)
resource "azurerm_nat_gateway" "ngw" {
  name                = "nat-gateway"
  location            = var.location
  resource_group_name = var.resource_group_name
  sku_name            = "Standard"
}

# 3. Lo enlazamos a la IP
resource "azurerm_nat_gateway_public_ip_association" "nat_assoc" {
  nat_gateway_id       = azurerm_nat_gateway.ngw.id
  public_ip_address_id = azurerm_public_ip.nat_ip.id
}

# 4. Magia Pura: Lo conectamos solo a un eslabón (La Subnet Privada - Bloque 4.2.3)
resource "azurerm_subnet_nat_gateway_association" "private_nat_assoc" {
  subnet_id      = azurerm_subnet.private.id
  nat_gateway_id = azurerm_nat_gateway.ngw.id
}
```

## 4.4. Exportar los recursos (outputs.tf)
Exporta de inmediato las raíces `id` de todo: `vnet_id`, `subnet_pub_a_id`, `subnet_pub_b_id` y muy valioso: `subnet_priv_id`.

- Deja libre la salida.

## 5.3. Public NSG
- Crea otro de estos objetos
- Permite acceso irrestricto del Inbound en puertos *80 y 443 (HTTP/HTTPS)* generados por origen `*`.

## 5.4. Private App NSG
- Permite SSH Entrada (Destino: 22) **única y exclusivamente** del bloque de CIDR correspondiente a la Subred Pública A u originado desde la IP Privada del objeto que se creará de Jumpbox.
- Permite HTTP Inbound (Destino: 80) para que tu Aplicación escuche *solamente* origen de las dos Subneds Públicas (Quien porta los Load Balancers reales).
- Permite salida integral.

*(A diferencia de AWS que ata estos a nivel Instancia, Azure recomienda fuertemente atarlos a la Subned. Haz una asociación final de estos 3 objetos hacia tus IDs correspondientes `subnet_pub_a_id`, etc utilizando objetos de Terraform de tipo `azurerm_subnet_network_security_group_association`)*

# 7. Módulo: Jumpbox (Bastión)
Este es un humilde PC expuesto de forma predecible con SSH, y es la cabina de acceso de cristal a tus ambientes.

## 7.1 Requerimentos Azure Nativos
Para instanciarlo requieres declarar primero `azurerm_public_ip` dinámica para conectarte a su cara, y la tarjeta de red interna real `azurerm_network_interface` pegada a la subnet pública.
**Importante**: Atribuye tu NSG creado hace unos momentos para blindar la NIC de los atacantes asiáticos y no dejar huecos.

## 7.2 Instanciar Computo Nativo en Azure
> [azurerm_linux_virtual_machine](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/linux_virtual_machine)

- tamaño: `Standard_B1s`.
- amárralo a la llave SSH que hicimos en el Step 3. (Pide por parámetro la string the llave publica `tls_private_key.privateKey.public_key_openssh`)
- usa Canonical UbuntuLTS de imagen de arranque interno.
- Configurar Etiqueta dinámica: Name = `"Jumpbox-${var.lab_name}"`

# 9. Módulo: Load Balancer
Mientras AWS depende pesadamente de su ALB genérico, implementaremos el robustazo Core en Azure LB para despachos L4 (O App Gateway si necesitas certificados SSL y cookies en el futuro - no abordado).

## 9.1 Base del Balanceador Frontal
```tf
# Cara del Sistema Final (Frontal de todo el Laboratorio)
resource "azurerm_public_ip" "lb_ip" {
  name                = "lb-public-ip"
  location            = var.location
  resource_group_name = var.resource_group_name
  allocation_method   = "Static"
  sku                 = "Standard"
  domain_name_label   = "wellarc-${lower(var.lab_name)}" # Para lograr apuntarlo por DNS eventualmente
}

# Bloque Administrativo Interno
resource "azurerm_lb" "web_lb" {
  name                = "web-load-balancer"
  location            = var.location
  resource_group_name = var.resource_group_name
  sku                 = "Standard"

  frontend_ip_configuration {
    name                 = "PublicIPAddress"
    public_ip_address_id = azurerm_public_ip.lb_ip.id
  }
}
```

## 9.2 Reglas y Entrenadores de Carga
Este balanceador no direcciona solo. Hay que decirle que hay atrás, como confirmar que no estén muertas, y que pasa cuando les hablan.
```tf
# El "Charco" o piscina de direcciones IP privadas muertas/vivas que aguantarán la carga.
resource "azurerm_lb_backend_address_pool" "bpepool" {
  loadbalancer_id = azurerm_lb.web_lb.id
  name            = "BackEndAddressPool"
}

# Constante de vida (¿Siguen vivas las VM privadas?)
resource "azurerm_lb_probe" "lb_probe" {
  loadbalancer_id = azurerm_lb.web_lb.id
  name            = "http-probe-80"
  port            = 80
}

# La regla de oro (Lo que caiga publicamente al 80, pásalo por tunel internamente al 80 a IPs privadas validas)
resource "azurerm_lb_rule" "lb_rule" {
  loadbalancer_id                = azurerm_lb.web_lb.id
  name                           = "http-forwarding-80"
  protocol                       = "Tcp"
  frontend_port                  = 80
  backend_port                   = 80
  frontend_ip_configuration_name = "PublicIPAddress"
  backend_address_pool_ids       = [azurerm_lb_backend_address_pool.bpepool.id]
  probe_id                       = azurerm_lb_probe.lb_probe.id
}
```

# 10. Módulo: Virtual Machine Scale Sets (Autoscale)
Este es el ente omnipotente que creara infinitos clones de código reaccionando al entorno en vivo sin que tú le administres hardware al detalle de IPs.

## 10.1 Escalar el clúster (`vmss.tf`)
Crea el marco técnico base del set atado al pool de tu Load Balancer.
```tf
resource "azurerm_linux_virtual_machine_scale_set" "vmss" {
  name                = "app-vmss-scaleset"
  resource_group_name = var.resource_group_name
  location            = var.location
  sku                 = "Standard_B1s"
  instances           = 1 # Arrancaremos de apoco en reposo.
  admin_username      = "adminuser"
  # Inyecta tu llave publica aquí nuevamente o no las podrás operar para debugging

  source_image_reference {
    publisher = "Canonical"
    offer     = "0001-com-ubuntu-server-jammy"
    sku       = "22_04-lts"
    version   = "latest"
  }

  os_disk {
    caching              = "ReadWrite"
    storage_account_type = "Standard_LRS"
  }

  network_interface {
    name    = "vmss-cloned-nic"
    primary = true

    ip_configuration {
      name      = "internal-dyn"
      primary   = true
      subnet_id = var.private_subnet_id  # <----- Aislado por subnet sin internet entrante
      load_balancer_backend_address_pool_ids = [var.lb_backend_pool_id]  # <---- Atado a que el balanceador te dispare
    }
  }

  # Corre la aplicación de contenedores en el primer Nano-segundo que prende.
  custom_data = filebase64("./config/app.sh")
}
```

## 10.2 Autoscale
Dile a Azure que mire las gráficas de esa máquina y ejecute acción coercitiva sin ti:
```tf
resource "azurerm_monitor_autoscale_setting" "autoscale_web" {
  name                = "autoscale-cpu-tracking"
  resource_group_name = var.resource_group_name
  location            = var.location
  target_resource_id  = azurerm_linux_virtual_machine_scale_set.vmss.id

  profile {
    name = "defaultEmergencyProfile"

    capacity {
      default = 1
      minimum = 1
      maximum = 2 # El techo que detiene el ataque a tu cartera (Gasto).
    }

    rule {
      metric_trigger {
        metric_name        = "Percentage CPU"
        metric_resource_id = azurerm_linux_virtual_machine_scale_set.vmss.id
        time_grain         = "PT1M"
        statistic          = "Average"
        time_window        = "PT5M" # Aguantando un asedio de 5 minutos al 50%+ CPU genera la alarma real
        time_aggregation   = "Average"
        operator           = "GreaterThan"
        threshold          = 50
      }
      scale_action {
        direction = "Increase"
        type      = "ChangeCount"
        value     = "1"
        cooldown  = "PT1M"
      }
    }
  }
}
```
