# 19. Práctica Final Módulos (Azure) <!-- omit in toc -->

# 1. Crear la carpeta PracticaFinal

# 2. Configurar el espacio de trabajo aisaldo
Vamos a aislar entornos. Crea dos workspaces:
- `dev`
- `prod`

# 3. Seleccionar el workspace "dev"
Colócate en el workspace `dev` para iniciar tu flujo local.

# 4. Crear terraform.tf, y asignar el provider
Escribe el archivo obligatorio fijando el proveedor principal `azurerm` (version `"~> 3.0"`).

# 5. Crear variables.tf en la raíz de tu proyecto
## 5.1. Variable de Ubicación (`location`)
- El bloque de la variable debe ser iterativo (tipo map) manejando los siguientes casos:
- Al ser llamado desde "dev"  = "East US"
- Al ser llamado desde "prod" = "West US"

## 5.2. Variable de Tamaño de Máquina Virtual (`vm_size`)
- Configura este map con los valores:
- "dev"  = "Standard_B1ls"
- "prod" = "Standard_B1s"

## 5.3. Variable de Bloque de Red Principal (`vnet_cidr`)
- Configura este map con los valores:
- "dev"  = "10.0.0.0/24"
- "prod" = "10.0.0.0/24"

## 5.4. Variable de Subred (`subnet_cidr`)
- Configura este map con los valores:
- "dev"  = "10.0.0.0/28"
- "prod" = "10.0.0.0/28"

## 5.5. Variable de Etiquetas Comunes (`tags`)
- Un mapa estático universal
- "owner" = "NOMBRE_DE_ESTUDIANTE"

# 6. Crear provider.tf
- Declara la carga del `azurerm` vacío con el preceptivo bloque interno `features {}`.

# 7. Separar lógicas en módulos (Crear la carpeta `./network`):
Agrega los siguientes archivos en esa nueva carpeta:
## 7.1. variables.tf
- Debe aceptar las variables recabadas del exterior: `vnet_cidr`, `subnet_cidr`, `tags`, y una para la locación final que llamaremos `res_location`.
- Estas variables vacías **NUNCA** deben tener valor `default`. Esperan parámetros.

## 7.2. base.tf (La Arquitectura Plomada)
### 7.2.1. Crea el Grupo Base (`azurerm_resource_group`)
- Asignar nombre dinámico: `"rg-${terraform.workspace}-final"`
- Asignar el location de la variable `res_location` correspondiente.

### 7.2.2. Crea la Red Virtual Principal (`azurerm_virtual_network`)
- Asígnale el entorno de direcciones extraído desde `vnet_cidr`.
- Adjúntala al grupo de recursos creado en el bloque anterior (utilizando recursión).
- Agrégale una sección the Tags obligatoria conteniendo:
  - owner = `var.tags["owner"]`
  - Name  = `"VNet-${terraform.workspace}"`

### 7.2.3. Crea la Subred (`azurerm_subnet`)
- Asegúrate que nazca atada a tu Red y Grupo previamente construidos.
- Extiende su rango desde la variable `subnet_cidr`.

## 7.3. outputs.tf
- Extrae el ID nominal del Resource Group (`rg_name`) recién creado, para exponerlo a otros modulos.
- Extrae y exporta el Id de tu Subnet.

# 8. Unirlo todo: El archivo main.tf Raíz
## 8.1. Invocar el Módulo Network
- Llama e inserta tu carpeta `./network`
- Abastécela forzando un mapeo de la función `lookup(...)` dictando a quien se escoge basándose en el estado de `terraform.workspace`.

# 9. Separar Servidores y Componentes Físicos (Crear la carpeta `./infra`):
Entra a ella y provee sus tres documentos:
## 9.1. variables.tf (Del Módulo Infra)
- Deben existir aquí las expectativas paramétricas: `vm_size`, `subnet_id`, `rg_name`, `res_location`, `tags`.
- No colocar campos `default`.

## 9.2. vm.tf (Configuración de Computo de Azure)
### 9.2.1. Configuración de Puntos de Red (`azurerm_public_ip` / `azurerm_network_interface`)
- Crea y asocia de forma dinámica una IP Pública genérica.
- Empata de forma vital el `subnet_id` proveniente del módulo A con una interfaz de red genérica (Network Interface - NIC).

### 9.2.2. Levantar El Servidor (`azurerm_linux_virtual_machine`)
- Atarla a tu `rg_name` y `res_location`.
- Dotarla del tipo de tamaño computacional según dicte `vm_size`.
- Asociar el ID recien extraido de tu Network Interface (NIC).
- (Punto libre) Utiliza UbuntuServer 22.04 LTS (Bloque `source_image_reference`) configurado con usuario "adminuser" y llaves/contraseñas libres para aprovisionar.
- Configurar Etiqueta dinámica: Name = `"${terraform.workspace}-vm"`

## 9.3. outputs.tf (Del Módulo Infra)
- Revela a los demás cuál es la IP Pública asignada
- Opcionalmente revela el ID final del servidor instanciado (`id`).

# 10. Completar el Círculo de Integración (Volver a la Raíz / main.tf principal)
## 10.1. Invocar el Módulo de Infraestructura (`./infra`)
- Declara la carga del segundo módulo.
- Alimenta `subnet_id` y `rg_name` apuntando la mira técnica explícitamente a las salidas correspondientes del primer bloque invocado (`module.network.xxxxx`).
- Cruza vía `lookup` las variables dependientes del workspace.

## 10.2. Crear tu Output definitivo en la ventana General (output.tf raíz)
- Muestra a la consola principal de tu terminal al finalizar el comando la `ip_publica` consumiendo la salida anidada de `module.infra.XXXXXXXX`.
