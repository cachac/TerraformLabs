# 10b. Funciones en Terraform (Opcional) <!-- omit in toc -->

Este laboratorio tiene como objetivo explorar el uso de las funciones integradas de Terraform, las expresiones de bucles (`for`), el uso de condicionales ternarios y las reglas de validación en variables de entrada.

> [!IMPORTANT]
> **Prerrequisito**: Este laboratorio crea recursos reales en AWS (S3 y EC2). Es necesario contar con credenciales de AWS configuradas en el entorno local (`aws configure`) para poder ejecutar `terraform apply`.

---

# 1. Explorar la consola interactiva de Terraform

Terraform cuenta con una consola interactiva para probar funciones, evaluar expresiones y validar variables antes de escribirlas en el código.

1.1. Iniciar la consola en la terminal:
```bash
terraform console
```

1.2. Probar las siguientes funciones estándar escribiéndolas en el prompt y presionando `Enter`:
```tf
# Redondeo hacia abajo (matemática)
floor(5.9)

# Formateo de texto
title("titulo-principal")
format("Hello, %s!", "nombre")
replace("1 + 2 + 3", "+", "-")

# Encontrar el valor máximo
max(5, 12, 9)

# Concatenar elementos de una lista con un separador
join("- ", ["var1", "var2", "var3"])
```

1.3. Para salir de la consola interactiva, presionar `Ctrl + C` o escribir `exit`.

# 2. Configurar el proyecto local

2.1. Crear un directorio llamado `funciones/` y ubicarse dentro de él.

2.2. Crear el archivo `providers.tf` para declarar el proveedor de AWS:
```tf
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 4.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}
```

# 3. Manipular strings y cargar archivos en S3

Se utilizará una lista de rutas de archivos para cargarlos dinámicamente en un bucket de S3, modificando sus nombres con la función `substr`.

3.1. Crear el archivo `variables.tf` para definir el nombre del bucket y los archivos a procesar:
```tf
variable "bucket_name" {
  type        = string
  description = "Nombre único para el bucket de S3"
  default     = "terraformlabs-test-bucket-100" # Se recomienda cambiar este valor por uno único
}

variable "media_files" {
  type    = set(string)
  default = [
    "../../assets/demo_func01.png",
    "../../assets/demo_func02.jpg",
    "../../assets/demo_func03.png",
  ]
}
```

3.2. Crear el archivo `main.tf` para declarar el bucket y el cargador de objetos:
```tf
resource "aws_s3_bucket" "media" {
  bucket = var.bucket_name
}

resource "aws_s3_object" "upload_media" {
  for_each = var.media_files

  bucket = aws_s3_bucket.media.id
  source = each.value
  
  # La función substr extrae una subcadena del texto: substr(cadena, inicio, longitud)
  # "../../assets/demo_func01.png" -> se convierte en "assets/demo_func01.png"
  key    = substr(each.value, 6, 23)
}
```

3.3. Crear el archivo `outputs.tf` para generar las URLs de los objetos subidos utilizando un bucle `for` de Terraform:
```tf
output "bucket_urls" {
  description = "Direcciones S3 de los objetos cargados"
  value = {
    for k, v in aws_s3_object.upload_media : k => "${v.bucket}.s3.amazonaws.com/${v.key}"
  }
}
```

# 4. Probar y validar la manipulación en la consola

Antes de aplicar los cambios, se puede validar cómo opera la función `substr` y los bucles `for` en la consola interactiva:

4.1. Iniciar la consola:
```bash
terraform console
```

4.2. Ejecutar las siguientes pruebas:
```tf
# Comprobar la extracción del nombre del archivo:

# Probar la conversión de todos los elementos a mayúsculas:
[for val in var.media_files : upper(val)]
```

4.3. Escribir `exit` para salir.

# 6. Reglas de validación en variables de entrada

Terraform permite definir reglas personalizadas para validar los parámetros antes de que sean aplicados a la infraestructura.

6.1. Crear el archivo `variables_instance.tf` con reglas de validación avanzadas:
```tf
variable "instance_name" {
  type        = string
  description = "Nombre para identificar la instancia EC2"

  validation {
    # La condición debe cumplirse (evaluar a true) para que la variable sea válida
    condition     = length(var.instance_name) > 5
    error_message = "El nombre de la instancia debe ser mayor a 5 caracteres."
  }
}

variable "instance_size" {
  type        = string
  description = "Tamaño de la instancia de cómputo (opciones: nano, micro)"

  validation {
    condition     = var.instance_size == "nano" || var.instance_size == "micro"
    error_message = "El tamaño de la instancia seleccionado no es válido. Las opciones son: nano o micro."
  }
}

variable "instance_type_nano" {
  type    = string
  default = "t3a.nano"
}

variable "instance_type_micro" {
  type    = string
  default = "t3a.micro"
}
```

# 7. Crear un recurso usando condicionales ternarios

Se creará una instancia EC2 cuyo tamaño dependerá de la decisión del usuario tomada a través de la variable `instance_size`.

7.1. Agregar el siguiente bloque al archivo `main.tf`:
```tf
resource "aws_instance" "server" {
  # Nota: El ID de la AMI y de la Subred son ejemplos de us-east-1.
  # Puede ser necesario actualizarlos con valores de su propia cuenta.
  ami           = "ami-033b95fb8079dc481"
  subnet_id     = "subnet-010205a1dea6ff7fc"

  # Condicional ternario: (condición ? si_verdadero : si_falso)
  instance_type = var.instance_size == "nano" ? var.instance_type_nano : var.instance_type_micro

  tags = {
    Name = var.instance_name
  }
}
```

7.2. Agregar la siguiente salida en `outputs.tf` para verificar el tipo de instancia creada:
```tf
output "instance_type_created" {
  description = "Tipo de instancia de cómputo asignado en AWS"
  value       = aws_instance.server.instance_type
}
```
