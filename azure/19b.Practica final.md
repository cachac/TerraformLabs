# 19. Práctica Final Módulos (Azure) <!-- omit in toc -->

# 1. Crear la carpeta PracticaFinal

# 2. Configurar el workspaces
Vamos a aislar entornos. Crea dos workspaces:
- `dev`
- `prod`

# 3. Seleccionar el workspace "dev"
Colócate en el workspace `dev` para iniciar tu flujo local.

# 4. Crear terraform.tf, y asignar el provider
Escribe el archivo obligatorio fijando el proveedor principal `azurerm` (version `"~> 3.0"`).

# 5. Crear variables.tf en la raíz del proyecto
## 5.1. Variable de region (`location`)
- Configura este map con los valores:
- "dev"  = "East US"
- "prod" = "West US"

## 5.2. Variable de Tamaño de Máquina Virtual (`vm_size`)
- Configura este map con los valores (modificar los valores de acuerdo a la disponibilidad de la región):
- "dev"  = "Standard_B1ls"
- "prod" = "Standard_B1s"

## 5.3. Variable de Bloque de Red Principal (`vnet_cidr`)
- Configura este map con los valores:
- "dev"  = "10.0.0.0/24"
- "prod" = "10.0.0.0/24"

## 5.4. Variable de Subred (`subnet_cidr`)
- Configura este map con los valores:
- "dev"  = "10.0.0.0/28"
- "prod" = "10.0.0.0/28"

## 5.5. Variable de Etiquetas Comunes (`tags`)
- Estos tags deben ser asignados a todos los recursos:
- "owner" = "NOMBRE_DE_ESTUDIANTE"

# 6. Crear provider.tf
- Declara la carga del `azurerm` vacío `features {}`.

# 7. Separar lógicas en módulos (Crear la carpeta `./network`):
Agrega los siguientes archivos en esa nueva carpeta:
## 7.1. variables.tf
- Debe aceptar las variables INPUT: `vnet_cidr`, `subnet_cidr`, `tags`, y una para la región que llamaremos `res_location`.
- Estas variables vacías **NUNCA** deben tener valor `default`. Los valores se los daremos desde la raís (main.tf).

## 7.2. rg.tf
### 7.2.1. Crea el Resource Group (`azurerm_resource_group`)
- Asignar nombre dinámico: `"rg-${terraform.workspace}-final"`
- Asignar el location de la variable `res_location` correspondiente.

## 7.3. vnet.tf
### 7.3.1. Crea la Red Virtual Principal (`azurerm_virtual_network`)
- Asignar el `vnet_cidr`.
- Asignar al resource group creado en el bloque anterior.
- Agrégale una sección the Tags obligatoria conteniendo:
  - owner = `var.tags["owner"]`
  - Name  = `"VNet-${terraform.workspace}"`

### 7.3.2. Crea la Subnet (`azurerm_subnet`)
- Asignar la vnet y resource group.
- Asignar su rango de IP desde la variable `subnet_cidr`.

## 7.4. outputs.tf
- Extrae el ID del Resource Group (`rg_name`), para exponerlo a otros modulos.
- Extrae y exporta el Id de tu Subnet.

# 8. Unirlo todo: El archivo main.tf Raíz
## 8.1. Invocar el Módulo Network
- Crear el módulog network con el source a `./network`
- Asignar las variables requeridas usando `lookup` para el workspace actual.

# 9. Crear máquinas virtuales (Crear la carpeta `./infra`):
## 9.1. variables.tf (Del Módulo Infra)
- Definir las variables: `vm_size`, `subnet_id`, `rg_name`, `res_location`, `tags`.
- Estas variables vacías **NUNCA** deben tener valor `default`. Los valores se los daremos desde la raís (main.tf).

## 9.2. vm.tf
### 9.2.1. Configuración de Red (`azurerm_public_ip` / `azurerm_network_interface`)
- Crear una IP Pública.
- Asignar el `subnet_id` proveniente del módulo `network` a la interfaz de red.

### 9.2.2. Servidor (`azurerm_linux_virtual_machine`)
- Asignar el resource group `rg_name` y `res_location`.
- Asignar el tamaño `vm_size`.
- Asociar el ID del Network Interface (paso anterior).
- Utiliza UbuntuServer 22.04.
- tags:
  - Name = `"${terraform.workspace}-vm"`
  - owner = `var.tags["owner"]`

## 9.3. outputs.tf (Del Módulo Infra)
- IP Pública
- ID final del servidor instanciado (`id`).

# 10. Llamar el módulo Infra (Volver a la Raíz / main.tf principal)
## 10.1. Invocar el Módulo de Infraestructura (`./infra`)
- Asignar el source.
- Asignar las variables necesarias usando `lookup`.

## 10.2. Crear tu Output en la carpeta root (output.tf raíz)
- Muestra a la consola principal de tu terminal al finalizar el comando la `ip_publica` y el `id` del servidor.

# 11. Aplicar (Workspace Dev)
- Revisar el plan.
- Ejecutar un `terraform apply`.
- Revisar los resultados en Azure y la IP publica del Output.

# 12. Workspace Prod
- Cambiar al workspace "prod".
- Ejecutar un `terraform apply`.
- Revisar los resultados en Azure y la IP publica del Output.

# 13. CheckPoint
- Ejecutar `terraform state list` en ambos workspaces y revisar resultados.

- Genera un `azurerm_network_security_group` permitiendo entradas exclusivas de Tráfico al puerto `22` (SSH) y `80` (HTTP).
- Conectar la regla al Network Interface principal.
