# 12. Práctica #1 <!-- omit in toc -->

Esta práctica tiene como objetivo consolidar los conocimientos adquiridos sobre variables, atributos, proveedores, dependencias y recursos de nube (AWS) mediante un ejercicio guiado y estructurado.

---

# Objetivos de la práctica
1. Generar una clave SSH privada de forma local utilizando Terraform.
2. Configurar el proveedor de AWS.
3. Crear un bucket de S3 privado de forma dinámica y cargar la clave SSH en él.
4. Crear una red virtual (VPC) en AWS con parámetros específicos.
5. Imprimir las salidas necesarias para verificar la creación de recursos.

# 2. Generar una clave SSH local

Se debe generar un par de claves criptográficas SSH (llave privada) mediante Terraform utilizando el proveedor `tls`.

2.1. Crear el archivo `variables.tf` para definir el nombre del archivo de la llave y el nombre del bucket de S3. Ambos valores se deben solicitar desde la terminal (sin valores por defecto):

```tf
variable "key_name" {
  type        = string
  description = "Nombre del archivo pem que guardará la clave SSH privada"
}

variable "bucket_name" {
  type        = string
  description = "Nombre único a nivel global para el bucket de S3"
}

variable "student_name" {
  type        = string
  description = "Nombre del estudiante para identificar la VPC"
}
```
> [!TIP]
> Para repasar cómo declarar variables sin valor por defecto, consultar la guía de [Variables Input](file:///Users/car/Documents/Code/terraformlabs/05.Variables%20Input.md).

2.2. Crear el archivo `main.tf` y configurar la generación de la clave SSH y su almacenamiento en un archivo local:

```tf
# Generación de la clave criptográfica privada
resource "tls_private_key" "ssh_key" {
  algorithm = "RSA"
  rsa_bits  = 4096
}

# Almacenamiento de la clave privada en un archivo local
resource "local_sensitive_file" "ssh_key_file" {
  content  = tls_private_key.ssh_key.private_key_pem
  filename = "${var.key_name}.pem"
}
```
> [!TIP]
> Para repasar el proveedor `tls` y el recurso `local_sensitive_file`, consultar la guía de [Atributos](file:///Users/car/Documents/Code/terraformlabs/06.Atributos.md).

# 3. Configurar el proveedor de AWS

3.1. En el archivo `main.tf`, añadir el bloque de proveedores requeridos y la configuración del proveedor de AWS:

```tf
terraform {
  required_providers {
    tls = {
      source  = "hashicorp/tls"
      version = "~> 4.0"
    }
    local = {
      source  = "hashicorp/local"
      version = "~> 2.0"
    }
    aws = {
      source  = "hashicorp/aws"
      version = "~> 4.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}
```
> [!TIP]
> Para repasar la inicialización y proveedores de AWS, consultar la guía de [AWS](file:///Users/car/Documents/Code/terraformlabs/11a.AWS.md).

# 4. Almacenamiento seguro en S3

Se debe crear un bucket de S3 privado en AWS y subir el archivo de clave criptográfica generado.

4.1. Crear el bucket en `main.tf`:
```tf
resource "aws_s3_bucket" "secure_storage" {
  bucket = var.bucket_name
}
```

4.2. Cargar la clave SSH privada en el bucket creado:
```tf
resource "aws_s3_object" "upload_key" {
  bucket = aws_s3_bucket.secure_storage.id
  key    = "${var.key_name}.pem"
  source = local_sensitive_file.ssh_key_file.filename
}
```

> [!IMPORTANT]
> **Evitar advertencias (Warnings)**:
> No utilizar el recurso antiguo `aws_s3_bucket_object`, ya que está obsoleto (*deprecated*). La documentación oficial indica utilizar el recurso actualizado **`aws_s3_object`**.

# 5. Crear una VPC (Red Virtual) en AWS

Investigar y declarar una red virtual VPC con los siguientes atributos:
* **CIDR Block**: `10.0.0.0/16`
* **Etiquetas**: Debe incluir la etiqueta `Name` con el valor `lab-NOMBRE_ESTUDIANTE` (haciendo uso de la variable `student_name`).

5.1. Agregar el recurso de VPC en `main.tf`:
```tf
resource "aws_vpc" "lab_vpc" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "lab-${var.student_name}"
  }
}
```

---

# 6. Definir salidas (Outputs)

6.1. Crear el archivo `outputs.tf` para mostrar los resultados de la implementación:
* El ID de la VPC creada.
* La URL directa del objeto S3 cargado.

```tf
output "vpc_id" {
  description = "El ID de la VPC creada en AWS"
  value       = aws_vpc.lab_vpc.id
}

output "ssh_key_s3_url" {
  description = "La URL del objeto S3 que almacena la clave privada"
  value       = "${aws_s3_bucket.secure_storage.bucket}.s3.amazonaws.com/${aws_s3_object.upload_key.key}"
}
```

# 7. Ejecutar y verificar la práctica

7.1. Inicializar el entorno:
```bash
terraform init
```

7.2. Ejecutar la planificación:
```bash
terraform plan
```

7.3. Aplicar los cambios ingresando los valores solicitados para las variables:
```bash
terraform apply
```

7.4. **Autoevaluación (Checkpoint)**:
Una vez completado con éxito, se pueden verificar los recursos en la nube mediante AWS CLI ejecutando los siguientes comandos en la terminal (reemplazando por los nombres asignados):

```bash
# Validar que el archivo se subió al bucket S3:
aws s3api list-objects --region=us-east-1 --bucket=<TU_NOMBRE_DE_BUCKET>

# Validar que la VPC existe con el ID generado:
aws ec2 describe-vpcs --region=us-east-1 --vpc-ids=<TU_VPC_ID_GENERADO>
```
