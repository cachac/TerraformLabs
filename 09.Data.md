# 09. Datasource <!-- omit in toc -->

> [!NOTE]
> **¿Qué es un Datasource?**
> Un bloque `data` (fuente de datos) permite a Terraform consultar y consumir información externa o recursos que ya existen en la infraestructura (ya sea creados manualmente, por otros proyectos de Terraform o por la propia nube). A diferencia de los bloques `resource`, los datasources son de **solo lectura** y no crean ni modifican infraestructura por sí mismos.

# 2. Crear archivo main.tf

```tf
data "local_file" "os" {
  filename = "/etc/os-release"
}

output "os-version" {
  value = data.local_file.os.content
}
```

# 3. Inicializar y aplicar
La salida mostrará el contenido del archivo local `/etc/os-release`.

# 5. Ejemplos de uso en la nube (Mundo Real)
En proyectos reales, los datasources se utilizan para consultar recursos de nube existentes y vincularlos dinámicamente.

### Ejemplo en AWS: Buscar la AMI más reciente de Ubuntu
En lugar de escribir el ID de la AMI manualmente en el código (el cual cambia constantemente), se puede buscar dinámicamente de la siguiente manera:

```tf
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"] # ID oficial de Canonical

  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-*"]
  }

  filter {
    name   = "virtualization-type"
    values = ["hvm"]
  }
}

# Uso del datasource para crear una instancia EC2:
resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id
  instance_type = "t3.micro"
}
```

### Ejemplo en AWS: Consultar una VPC existente por etiquetas (Tags)
Si se desea desplegar recursos dentro de una red (VPC) que ya fue creada por otro equipo o proyecto, se puede consultar mediante etiquetas:

```tf
data "aws_vpc" "seleccionada" {
  filter {
    name   = "tag:Environment"
    values = ["Production"]
  }
}

# Uso del ID de la VPC para crear un Grupo de Seguridad:
resource "aws_security_group" "web_sg" {
  name   = "sg_web"
  vpc_id = data.aws_vpc.seleccionada.id
}
```
