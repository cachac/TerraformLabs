# 13. Estado Remoto (Azure) <!-- omit in toc -->

# 1. Crear la carpeta remote/

# 2. Crear el archivo main.tf para crear un archivo local

```tf
resource "local_file" "state_demo" {
  filename = "local.txt"
  content  = "Archivo de estado: local"
}
```

# 3. Crear los recursos de Almacenamiento en Azure
```tf
resource "azurerm_resource_group" "remote_state" {
  name     = "terralabs-state-rg"
  location = "East US"
}

resource "azurerm_storage_account" "remote_state" {
  name                     = "terralabsstateremoto01" # el nombre debe ser unico, solo minúsculas.
  resource_group_name      = azurerm_resource_group.remote_state.name
  location                 = azurerm_resource_group.remote_state.location
  account_tier             = "Standard"
  account_replication_type = "LRS"

  tags = {
    Environment = "dev"
  }
}

resource "azurerm_storage_container" "remote_state" {
  name                  = "tfstate"
  storage_account_name  = azurerm_storage_account.remote_state.name
  container_access_type = "private"
}
```

# 4. Crear el provider.tf
```tf
provider "azurerm" {
  features {}
}
```

# 5. Crear el archivo terraform.tf especificando la versión de Azure
```tf
terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.0"
    }
  }
}
```

# 6. Iniciar y Aplicar
```vim
terraform init
terraform apply
```
> Se crea el archivo de estado local: terraform.tfstate
>
> Se crean los recursos de almacenamiento (Resource Group, Storage Account y Container) en Azure.


# 7. Agregar la configuración del Backend "azurerm" al archivo terraform.tf
```tf
terraform {
  # ... (mantener bloque required_providers existente)

  backend "azurerm" {
    resource_group_name  = "terralabs-state-rg"         # Nombre de tu Resource Group
    storage_account_name = "terralabsstateremoto01"     # Nombre único de tu Storage Account
    container_name       = "tfstate"                    # Nombre del contenedor
    key                  = "terraform.tfstate"          # Nombre del archivo en Azure
  }
}
```

# 8. Inicializar el backend y migrar el estado local a la nube
```vim
terraform init
```

Resultado:
> Initializing the backend...
> Do you want to copy existing state to the new backend?

> Successfully configured the backend "azurerm"! Terraform will automatically
> use this backend unless the backend configuration changes.

# 9. Validar el contenedor con el contenido del terraform.tfstate subido a Azure
```vim
az storage blob list --account-name terralabsstateremoto01 --container-name tfstate --output table
```

# 10. Cambiar el contenido del recurso local_file y aplicar.

> Los cambios han sido aplicados automáticamente al archivo de estado remoto (en Azure Blob Storage).


# 11. Comandos de estado
```vim
terraform state list

terraform state show local_file.state_demo

terraform state pull

terraform state rm -dry-run <resource>.<name>
```

# 12. Destruir
La cuenta de almacenamiento y el contenedor están en uso porque guardan el estado actual (`terraform.tfstate`). Terraform no podrá destruir todo el entorno limpiamente a menos que vuelva a tener posesión local de ese archivo primero.

Debemos traer el estado de vuelta a local antes de poder destruir toda la infraestructura.

## 12.1. Eliminar o comentar la configuración del bloque `backend "azurerm"` en el archivo `terraform.tf`
## 12.2. Traer de vuelta el `terraform.tfstate` a la máquina local
```vim
terraform init -migrate-state
```

## 12.3. Destruir toda la infraestructura en Azure
```vim
terraform destroy -auto-approve
```
