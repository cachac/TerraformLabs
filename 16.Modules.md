# 3. En /modules/file-module crear los archivos:

local.tf
```tf
resource "local_file" "module-demo" {
  filename = var.in-filename
  content  = "Este es mi primer módulo"
}
```

variable.tf
```tf
variable "in-filename" {
  type = string
}
```

output.tf
```tf
output "out-filename" {
  value = local_file.module-demo.content
}
```

# 4. En /modules crear los archivos:

main.tf
```tf
locals {
  filename = "test.txt"
}

module "file-module" {
  source = "./file-module"
}
```

> [locals](https://learn.hashicorp.com/tutorials/terraform/locals)

# 9. Práctica
## 9.1. Cambiar el contenido de los dos archivos usando variables INPUT en el módulo
- test.txt = Este es el archivo #1 de <NOMBRE>
- dos.txt = Este es el archivo #2 de <NOMBRE>

## 9.2. Cambiar los permisos de los archivos usando variables INPUT en el módulo
- test.txt = 400
- dos.txt = 600

## 9.3. Crear un nuevo módulo para generar un string aleatorio
- Crear la carpeta `/modules/random` con sus archivos de configuración (`random.tf`, `variable.tf` y `output.tf`).
- Dentro de este módulo, definir un recurso `random_string` ([documentación oficial](https://registry.terraform.io/providers/hashicorp/random/latest/docs/resources/string)).
- La longitud del string (`length`) debe ser configurable a través de una variable INPUT del módulo.
- Exponer el string generado a través de una variable OUTPUT.
- Llamar al módulo `random` desde el `main.tf` raíz pasando la longitud deseada.

## 9.4. Utilizar el valor aleatorio en los archivos de los módulos `file-module` & `file-module-dos`
- Pasar el OUTPUT generado por el módulo `random` como parte del contenido en las llamadas de `file-module` y `file-module-dos`.
- El contenido de ambos archivos debe incluir el string aleatorio al final (ej. `Este es el archivo #1 de <NOMBRE> - <STRING_ALEATORIO>`).
- Ejecutar `terraform init` y `terraform apply` para verificar que ambos archivos contengan el mismo string aleatorio.
