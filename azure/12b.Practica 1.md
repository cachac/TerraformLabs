# 2. Crear una llave ssh local
- El nombre de la llave debe ser provista desde la terminal. (-var)

> [Mas información de Llaves SSH](https://registry.terraform.io/providers/hashicorp/tls/latest/docs/resources/private_key)

# 3. Configurar la máquina virtual con los accesos a Azure
Usar el método de configuración por terminal:
```vim
az login --use-device-code
```
> Sigue las instrucciones para autenticar tu cuenta en el navegador.

# 4. Solicitar al instructor permisos (Rol Contributor) en la suscripción de Azure

# 5. Configurar "provider" de Azure
> Configura el provider `azurerm` con su bloque `features {}`.

# 6. Subir la llave a un Storage Blob (Azure) *PRIVADO*
## 6.1. Recuerda que necesitas un Resource Group y un Storage Account primero.
## 6.2. El nombre del Storage Account debe ser un valor único (en minúsculas y sin guiones).
## 6.3. El nombre del contenedor de almacenamiento (container) debe ser provisto por una variable.

> [configuración Storage Account](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/storage_account)

> [configuración Storage Container](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/storage_container)

> [configuración Storage Blob](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/storage_blob)

# 7. Mostrar la salida del URL del archivo
- Crear un output para mostrar la propiedad `url` del blob que contiene tu llave SSH.

## 7.1. Chequear el blob con az cli
```vim
az storage blob list --account-name NOMBRE_DEL_STORAGE_ACCOUNT --container-name NOMBRE_DEL_CONTENEDOR --output table
```

# 8. Investigar. Crear una Virtual Network (VNet) con los siguientes atributos:
- nombre: lab-NOMBRE_ESTUDIANTE
- espacio de direcciones (address space): 10.0.0.0/16
- *(Nota: Ubícala en el mismo Resource Group que tu cuenta de almacenamiento)*

> [Mas info Virtual Network](https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs/resources/virtual_network)

# 9. Crear el output con el vnet-id
- Mostrar el `id` de la `azurerm_virtual_network`.

# 10. Checkpoint
- Validar con el instructor el Blob con la llave SSH privada subida
- Validar la VNet con los valores correctos
```vim
az storage blob list --account-name NOMBRE_DEL_STORAGE_ACCOUNT --container-name NOMBRE_DEL_CONTENEDOR --output table
az network vnet show --resource-group NOMBRE_DEL_RESOURCE_GROUP --name lab-NOMBRE_ESTUDIANTE --output table
```
